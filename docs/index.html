Architecture

How it works

Three steps. The middle one is the whole point — verification happens on Stellar, in a contract, using native cryptography.

01
Off-chain · Circom

Prove privately

A circuit proves the issuer's private balances sum to a public total, range-checking each to block the Mt. Gox negative-liability trick. Balances stay on the issuer's machine.

02
On-chain · Soroban

Verify on Stellar

The contract checks the Groth16 proof with Stellar's native BLS12-381 host functions (Protocol 25/26), then confirms the attested reserve covers the proven liabilities.

03
Public · anyone

Trust the verdict

Anyone reads SOLVENT / INSOLVENT from the chain. A tampered total makes the pairing check fail, so a lie can never be recorded as a valid attestation.
